
1. 系统：
  a. 数据类放在 app/core/model/memory.py 下面，比如一些 memory
  b. 对于一些 memfuse 需要的系统级别的配置，可以在 app/core/common/system_env.py 中的 SystemEnv 中配置相关的默认配置（现状：SystemEnv 在启动的时候会 load .env，意味着 SystemEnv 用于处理和管理环境）。
  c. System 新增 ENABLE_MEMFUSE，其它所有关于 memfuse 的配置，都要用 MEMFUSE_** 前缀。
  d. 不用 logging，而是直接 print（无色打印）。并且需要在 SystemEnv 添加一个参数 PRINT_MEMORY_LOG = False），来判断是否打印。
2. ReasonerMemory：
  a. 对于 Reasoner 的 memory，继承 ReasonerMemory叫做 MemFuseMemory。
  b. ReasonerMemory 需要添加一些 base methods（称为 A）：存储历史消息、读取记忆等这样的 methods，方法名根据实际情况确定。
3. MemoryService：
  a. memory service 放在 app/core/service/ 下，是 Singleton。
    ⅰ. MemoryService 管理一个字典：memories：Dict[job_id, Dict[operator_id, reasoner_memory]]（即，所有的 memory 存在这个字典中，未来再做持久化）。
    ⅱ. MemoryService 有一个方法 get_or_create_reasoner_memory(self, job_id: str, operator_id: str) -> ReasonerMemory，通过检测哪一类型的 memory 模块被 enable 了（比如 SystemEnv.ENABLE_MEMFUSE = true），则返回哪种 memory 形式（保底返回 BuiltinReasonerMemory）。
    ⅲ. 如果返回， get_or_create_operator_memory(self, job_id: str, operator_id: str) -> ReasonerMemory，通过检测哪一类型的 memory 模块被 enable 了（比如 SystemEnv.ENABLE_MEMFUSE = true），则返回哪种 memory 形式（保底返回 BuiltinReasonerMemory）。
    ⅳ. 因此，每次 Reasoner 调用的记忆模块都是 ReasonerMemory 或者它的子类。如果要实现热插拔，使用 get_or_create_memory() 即可。
4. Reasoner 使用记忆功能：
  a. 如果要修改，需要同时修改 Reasoner、DualModelReasoner、MonoModelReasoner。
  b. 去掉 Reasoner 中的 evaluate 方法，添加相关的调用记忆模块的「方法」（即，之前提到的 A），可能是叫 hook 什么的。
  c. 通过 MemoryService 的接口来使用记忆功能。
  d. 不应该修改 ReasonerService 的代码。
5. Operator 使用记忆功能：
  a. 看了代码，无需大修。
  b. 通过 MemoryService 的接口来使用记忆功能。 
6. 代码规范：
  a. 检测 mypy app，mypy test/example 没有报错，ruff run app，ruff run test/example 没有报错。行字数超过若干字数（ruff 会提示），可以使用 # noqa: E501
  b. 代码注释：小写开头，英文。尽量避免和代码出现在同一行，如果出现，则空两格再开始注释 #
